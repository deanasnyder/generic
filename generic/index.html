Preliminary Remarks Section:

This web page illustrates several HTML, CSS, and JavaScript techniques I currently like. Here's a partial list:

  • This standalone web app has no external dependencies - all referenced files are included in the project directory.
  • It implements a site-wide, light/dark color scheme strategy.
  • It populates an HTML5-specific head element correctly.
  • It uses a single global variable, "A", that contains all our functions and variables, thereby avoiding pollution of the JavaScript namespace.
  • It gets the browser-specific, best timer for profiling code performance.
  • It configures the mobile viewport for optimal display.
  • Its "init.css" stylesheet initializes CSS styles to the same vanilla state across all browsers by combining techniques of my own with those adapted from "reset.css" and "normalize.css".
  • It uses jQuery to manipulate the DOM.
  • It uses responsive design techniques to re-configure the page based on screen size.
  • It has a fixed header with a popup menu.
  • It animates scrolling to links on the same page links, compensating for the height of the fixed header.
  • It implements various assistive technologies that computer screen readers use to help visually-impaired users interact with web pages - for example: web page structural elements, menus, and focus.
  • All links and controls on this page can be manipulated using the keyboard; the mouse is not required.
  • Focus changes are animated so they are more noticeble. (You can traverse the keyboard focusable elements by pressing the tab key.)
  • The automated, modularized footer provides useful site information.
  • The HTML code validates with the W3C Markup Validation Service.
  • The CSS code validates with the W3C CSS Validation Service.

Semantic Elements Section:

We use semantic elements like header, nav, main, article, and section as aids for screen readers, web crawlers, and search engines in analyzing web page structure.
